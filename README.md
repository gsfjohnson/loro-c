# loro-c

C and C++ bindings for [Loro](https://github.com/loro-dev/loro), a high-performance CRDT
library. A thin Rust crate (`loro-c-api`) exposes a stable C ABI; a hand-written,
header-only C++20 layer (`loro/loro.hpp`) adds RAII and exceptions on top.

> **Status: core document model + all six containers + events + the advanced subsystems +
> rich text; advancing toward full `loro-ffi` parity.** The surface covers `LoroDoc`, all six
> containers (`Text` — including rich-text marks, deltas, and UTF-16/position helpers — `Map`,
> `List`, `MovableList`, `Tree`, `Counter`), events & subscriptions, and the advanced APIs
> (awareness, ephemeral store, undo manager, version vectors / frontiers / change ancestors,
> fractional index, JSON Path, commit hooks). Still outstanding — cursors, JSON-update sync &
> export modes, diff/patch, structured-value navigation, and the doc-utility/attribution long
> tail — are tracked in [pm/GAPS_PLAN.md](pm/GAPS_PLAN.md). The library installs as a
> relocatable CMake package (`find_package(loro)`) and ships a `pkg-config` file. A versioned
> shared library (`cdylib` + `SOVERSION`) is the one remaining packaging follow-up — today the
> install is a static archive. See [pm/CBINDGEN_PLAN.md](pm/CBINDGEN_PLAN.md) for the original
> milestone roadmap.

## Layout

```
include/loro/loro.h     C ABI header — generated by cbindgen, committed, source of truth
include/loro/loro.hpp   C++ RAII wrapper — hand-written, header-only
loro-c-api/             the Rust crate that implements the C ABI
cmake/                  package config + pkg-config templates; manual cargo->CMake fallback
examples/               basic_text, rich_text, sync_two_docs, subscribe_events
tests/                  CTest suite (C++ and pure-C) + tests/consumer (find_package smoke test)
.github/workflows/      CI: build/test matrix + install smoke test + cbindgen drift guard
```

## Building

Requires a C/C++ toolchain, CMake ≥ 3.22, and a Rust toolchain (`cargo`).

### Default (Corrosion)

Corrosion is fetched automatically and drives the cargo build:

```sh
cmake -S . -B build -DBUILD_TESTING=ON
cmake --build build
ctest --test-dir build --output-on-failure
```

### Manual cargo fallback

On hosts where Corrosion's absolute-path handling breaks — notably **MSYS2/CLANG64**, where
the only CMake is a POSIX build driving a native-Windows `cargo` — disable Corrosion and use
the bundled wrapper instead:

```sh
cmake -S . -B build -G Ninja \
  -DCMAKE_C_COMPILER=clang -DCMAKE_CXX_COMPILER=clang++ \
  -DLORO_USE_CORROSION=OFF \
  -DLORO_RUST_TOOLCHAIN=stable-x86_64-pc-windows-gnullvm
cmake --build build
ctest --test-dir build --output-on-failure
```

`LORO_RUST_TOOLCHAIN` is applied via `RUSTUP_TOOLCHAIN` so the cargo shim and the `rustc`
it spawns resolve to the same toolchain — required on a box whose default toolchain (e.g.
msvc) cannot link in this environment.

### Examples

```sh
cmake -S . -B build -DLORO_BUILD_EXAMPLES=ON   # (+ the flags above in manual mode)
cmake --build build
./build/examples/basic_text
./build/examples/sync_two_docs
./build/examples/subscribe_events
```

## Installing & consuming

Install the headers, the static archive, the CMake package files, and the `pkg-config` file
under a prefix:

```sh
cmake --install build --prefix /path/to/prefix
```

This produces:

```
<prefix>/include/loro/{loro.h,loro.hpp}
<prefix>/lib/libloro_c_api.a
<prefix>/lib/cmake/loro/{loroConfig.cmake,loroConfigVersion.cmake,loro-targets.cmake,…}
<prefix>/lib/pkgconfig/loro.pc
```

### From CMake (`find_package`)

```cmake
find_package(loro REQUIRED)            # add CMAKE_PREFIX_PATH=<prefix> if non-standard
add_executable(app main.cpp)
target_link_libraries(app PRIVATE loro::loro)
```

`loro::loro` carries the include path, the C++20 requirement, and the transitive link to the
Rust archive plus its platform system libraries — no manual `-l` flags needed.
(`tests/consumer/` is exactly this, used as the install smoke test in CI.)

### From pkg-config

```sh
c++ -std=c++20 main.cpp $(pkg-config --cflags --libs --static loro) -o app
```

Use `--static` so the system libraries the archive needs (`Libs.private`) are included.

## Usage

### C++

```cpp
#include <loro/loro.hpp>

loro::Doc doc;
loro::Text text = doc.get_text("greeting");
text.insert(0, "hello world");
doc.commit();

std::vector<std::uint8_t> snapshot = doc.export_snapshot();

loro::Doc restored;
restored.import(snapshot);
// restored.get_text("greeting").to_string() == "hello world"
```

Fallible calls throw `loro::Error` (carrying the `LoroStatus` and the library's last-error
message) instead of returning a status code. Subscribe to changes with
`doc.subscribe_root(cb)` / `doc.subscribe(container_id, cb)`; the returned `loro::Subscription`
unsubscribes on destruction (see `examples/subscribe_events.cpp`).

### C

```c
#include <loro/loro.h>

LoroDoc* doc = loro_doc_new();
LoroText* t = loro_doc_get_text(doc, "greeting", 8);
loro_text_insert(t, 0, "hello world", 11);
loro_doc_commit(doc);

LoroBytes snapshot = {0};
loro_doc_export_snapshot(doc, &snapshot);
/* ... loro_doc_import into another doc ... */
loro_bytes_free(snapshot);

loro_text_free(t);
loro_doc_free(doc);
```

Every fallible C function returns a `LoroStatus` (`LORO_OK == 0`); call
`loro_last_error_message()` for detail. Owned outputs (`LoroBytes`, handles) are freed with
their matching `loro_*_free` function.

## Lifetimes

A container handle (`loro::Text` / `LoroText*`) is a **strong co-owner** of its document's
state. It stays valid even after the parent `Doc` is freed, and the state is released only
once the doc *and every container obtained from it* are gone. Handles may be freed in any
order — there is no destruction-order hazard. (Both test suites exercise the
free-the-doc-before-the-text case.)

## Regenerating the C header

`include/loro/loro.h` is committed and is the source of truth; a normal build never
regenerates it. After changing the Rust API, regenerate it explicitly with the pinned
cbindgen CLI:

```sh
cmake --build build --target regenerate-header
```

CI runs the same cbindgen command and fails if the committed header is stale.
